# Writing a ros2_control controller plugin

A controller derives from `controller_interface::ControllerInterface`, declares the
interfaces it claims/reads, and computes commands in `update()`. It is loaded by the
controller_manager via `pluginlib` — it is a library, not an executable.

Package deps (`package.xml`): `controller_interface`, `hardware_interface`, `pluginlib`,
`rclcpp_lifecycle`, `realtime_tools`, `generate_parameter_library` (optional but preferred),
plus the message packages you use.

## Class skeleton

```cpp
// include/my_controllers/my_controller.hpp
#pragma once
#include <controller_interface/controller_interface.hpp>
#include <realtime_tools/realtime_buffer.hpp>
#include <std_msgs/msg/float64_multi_array.hpp>
#include "my_controllers/my_controller_parameters.hpp"  // generated by gen_param_lib

namespace my_controllers {

class MyController : public controller_interface::ControllerInterface {
 public:
  controller_interface::CallbackReturn on_init() override;

  controller_interface::InterfaceConfiguration command_interface_configuration() const override;
  controller_interface::InterfaceConfiguration state_interface_configuration() const override;

  controller_interface::CallbackReturn on_configure(
      const rclcpp_lifecycle::State & previous_state) override;
  controller_interface::CallbackReturn on_activate(
      const rclcpp_lifecycle::State & previous_state) override;
  controller_interface::CallbackReturn on_deactivate(
      const rclcpp_lifecycle::State & previous_state) override;

  controller_interface::return_type update(
      const rclcpp::Time & time, const rclcpp::Duration & period) override;

 private:
  std::shared_ptr<ParamListener> param_listener_;   // from generated header
  Params params_;
  realtime_tools::RealtimeBuffer<std::vector<double>> rt_command_;
  rclcpp::Subscription<std_msgs::msg::Float64MultiArray>::SharedPtr cmd_sub_;
};

}  // namespace my_controllers
```

## Interface configuration

Tell the controller_manager which hardware interfaces this controller claims (command) and
reads (state). `interface_configuration_type` is `INDIVIDUAL` (list names), `ALL`, or `NONE`.
Names are `"<joint>/<interface>"`; use the `hardware_interface::HW_IF_*` constants.

```cpp
controller_interface::InterfaceConfiguration
MyController::command_interface_configuration() const {
  controller_interface::InterfaceConfiguration cfg;
  cfg.type = controller_interface::interface_configuration_type::INDIVIDUAL;
  for (const auto & joint : params_.joints) {
    cfg.names.push_back(joint + "/" + hardware_interface::HW_IF_EFFORT);
  }
  return cfg;
}

controller_interface::InterfaceConfiguration
MyController::state_interface_configuration() const {
  controller_interface::InterfaceConfiguration cfg;
  cfg.type = controller_interface::interface_configuration_type::INDIVIDUAL;
  for (const auto & joint : params_.joints) {
    cfg.names.push_back(joint + "/" + hardware_interface::HW_IF_POSITION);
    cfg.names.push_back(joint + "/" + hardware_interface::HW_IF_VELOCITY);
  }
  return cfg;
}
```

After activation the claimed handles live in the base-class members `command_interfaces_`
and `state_interfaces_` (a `std::vector` of loaned interfaces), **ordered exactly as you
listed them above**. Cache indices in `on_activate` so `update()` does no lookups.

## Lifecycle bodies

```cpp
controller_interface::CallbackReturn MyController::on_init() {
  try {
    param_listener_ = std::make_shared<ParamListener>(get_node());
  } catch (const std::exception & e) {
    RCLCPP_ERROR(get_node()->get_logger(), "on_init failed: %s", e.what());
    return controller_interface::CallbackReturn::ERROR;
  }
  return controller_interface::CallbackReturn::SUCCESS;
}

controller_interface::CallbackReturn MyController::on_configure(
    const rclcpp_lifecycle::State &) {
  params_ = param_listener_->get_params();                     // typed, validated
  rt_command_.writeFromNonRT(std::vector<double>(params_.joints.size(), 0.0));
  cmd_sub_ = get_node()->create_subscription<std_msgs::msg::Float64MultiArray>(
      "~/commands", rclcpp::SystemDefaultsQoS(),
      [this](std_msgs::msg::Float64MultiArray::SharedPtr msg) {
        rt_command_.writeFromNonRT(msg->data);                 // non-RT writer
      });
  return controller_interface::CallbackReturn::SUCCESS;
}

controller_interface::CallbackReturn MyController::on_activate(
    const rclcpp_lifecycle::State &) {
  if (command_interfaces_.size() != params_.joints.size()) {
    return controller_interface::CallbackReturn::ERROR;        // wiring mismatch
  }
  return controller_interface::CallbackReturn::SUCCESS;
}
```

`update()` is the real-time path — see the RT contract in `../SKILL.md` and the full rules
in `.trellis/spec/robotics/cpp-performance.md`. Read state with
`state_interfaces_[i].get_value()`, write commands with `command_interfaces_[i].set_value(x)`,
and pull references from the `RealtimeBuffer` — no allocation, no locking, no logging.

## Parameters via generate_parameter_library

```yaml
# src/my_controller_parameters.yaml
my_controllers:
  joints:
    type: string_array
    default_value: []
    validation: { not_empty<>: [] }
  kp:
    type: double_array
    default_value: [100.0]
```

`CMakeLists.txt`: `generate_parameter_library(my_controller_parameters
src/my_controller_parameters.yaml)` then link `my_controller_parameters` into the controller
target. This generates `my_controller_parameters.hpp` with `ParamListener` + `Params`.

## pluginlib export

```cpp
// bottom of my_controller.cpp
#include "pluginlib/class_list_macros.hpp"
PLUGINLIB_EXPORT_CLASS(
    my_controllers::MyController, controller_interface::ControllerInterface)
```

```xml
<!-- my_controllers.xml -->
<library path="my_controllers">
  <class name="my_controllers/MyController"
         type="my_controllers::MyController"
         base_class_type="controller_interface::ControllerInterface">
    <description>PD effort controller.</description>
  </class>
</library>
```

```cmake
# CMakeLists.txt
add_library(my_controllers SHARED src/my_controller.cpp)
ament_target_dependencies(my_controllers controller_interface hardware_interface
                          pluginlib rclcpp_lifecycle realtime_tools)
target_link_libraries(my_controllers my_controller_parameters)
pluginlib_export_plugin_description_file(controller_interface my_controllers.xml)
install(TARGETS my_controllers LIBRARY DESTINATION lib)
```

## Controller manager YAML

```yaml
controller_manager:
  ros__parameters:
    update_rate: 1000  # Hz — the update() loop rate
    joint_state_broadcaster:
      type: joint_state_broadcaster/JointStateBroadcaster
    my_controller:
      type: my_controllers/MyController

my_controller:
  ros__parameters:
    joints: [joint1, joint2]
    kp: [120.0, 120.0]
```

Spawn after the manager is up: `ros2 run controller_manager spawner my_controller`
(and `joint_state_broadcaster`). Inspect with `ros2 control list_controllers` and
`ros2 control list_hardware_interfaces`.
